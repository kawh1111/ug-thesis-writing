# 写作经验总结：常见问题与解决方案

本文档总结了使用 Claude Code 辅助本科毕业论文写作过程中的常见问题和解决方案。

## 一、适用场景

- **论文类型**: 本科毕业论文（理工科）
- **写作工具**: Microsoft Word (.docx)
- **引用管理**: Zotero / EndNote
- **辅助工具**: python-docx, Word COM (win32com), python-pptx

## 二、常见问题与解决方案

### 问题1：写作顺序不当导致返工

**常见错误**: 先写绪论，再做实验，结果发现绪论内容与实际结果不匹配。

**推荐做法**: 先写方法和结果，再写绪论和讨论。
- 方法和结果依赖真实实验数据，必须先有数据才能写
- 绪论需要了解研究全貌后再写更准确
- 讨论需要结果支撑

### 问题2：DOCX 编辑破坏引用字段

**常见错误**: 直接用 python-docx 修改含 Zotero 引用的段落，导致引用丢失。

**推荐做法**: python-docx + Word COM 双引擎策略。
- python-docx 做段落级文本编辑（安全保留字段）
- Word COM 做目录刷新、图片替换等需要 Word 渲染的操作
- 编辑前后计数 Zotero 字段标记，验证未丢失
- 不要修改含 fldChar 的段落

### 问题3：格式修正效果不理想

**常见错误**: 直接用 python-docx 修改格式，发现很多段落格式读取不准确。

**推荐做法**:
- 先提取学校/学院格式要求，创建对比表
- 用 Word COM 读取实际渲染格式，再用 python-docx 修改
- 格式检查前先确认"以哪个标准为准"（学院要求 vs 学校模板可能冲突）
- 批量修正前必须在少量段落上测试

### 问题4：AIGC 降重反而提高检测率

**常见错误**: 用 AI 全文重写来降重，结果检测率不降反升。

**推荐做法**: 确定性替换，而非 AI 重写。
- AI 重写 AI 文本 = 叠加 AI 痕迹（困惑度更低，更容易被检测）
- 修改率必须超过 40%
- 句式重构（语序重组、长句拆分）比词汇替换更有效
- "XX了"模式是高风险信号，优先处理

### 问题5：导师批注处理时破坏格式

**常见错误**: 直接按批注修改，发现有些修改破坏了 Zotero 字段或公式对象。

**推荐做法**:
- 先分类批注类型（文字修改/结构调整/内容补充/格式修正）
- 含 Zotero 字段的段落不能直接修改文本，需特殊处理
- 结构调整前先备份，调整后验证目录和引用
- 生成修改对照表（原文 → 修改后 → 修改原因）便于导师复核

### 问题6：答辩准备不充分

**常见错误**: 先做 PPT 再写讲稿，导致内容逻辑混乱。

**推荐做法**:
- 先写讲稿确定内容逻辑，PPT 只是视觉呈现
- 准备 3 分钟压缩版（答辩可能被要求缩短时间）
- 准备 25+ 个 Q&A 问题，覆盖方法类/数据类/结果类/局限性类
- 关键数据要背熟

## 三、工具链推荐

| 工具 | 用途 | 备注 |
|------|------|------|
| python-docx | 文本编辑、格式修改 | 安全，但无法刷新目录 |
| Word COM | 目录刷新、图片替换 | 仅 Windows，需安装 Word |
| python-pptx | PPT 生成 | 需要模板 |
| Zotero / EndNote | 引用管理 | 字段脆弱，编辑时需小心 |
| Claude Code | 写作辅助、格式检查 | 配合自定义技能使用 |

## 四、典型时间线参考

| 阶段 | 参考时长 | 主要工作 |
|------|----------|----------|
| 项目初始化 | 1天 | 读取任务书，建立目录结构 |
| 文献综述 | 1-2周 | 检索、阅读、撰写综述 |
| 章节写作 | 2-4周 | 逐章撰写，反复修改 |
| 格式修正 | 2-3天 | 检查、对比、批量修正 |
| AIGC 降重 | 1-2天 | 扫描、替换、验证 |
| 导师修改 | 1-2周 | 提取批注、逐条处理 |
| 答辩准备 | 2-3天 | PPT、讲稿、Q&A |
| 终稿提交 | 1天 | 最终检查、打印、提交 |

## 五、核心原则

1. **先写后改**: 不要追求一次写完美，先完成再修改
2. **备份为王**: 每次操作前备份，避免不可逆错误
3. **字段保护**: 引用字段是论文的命脉，必须保护
4. **确定性编辑**: 避免 AI 全文重写，使用确定性替换
5. **格式先行**: 先确认标准，再批量修正
6. **讲稿优先**: 先写讲稿再做 PPT
7. **Q&A 充足**: 准备足够多的问题，覆盖所有可能被问到的方向
