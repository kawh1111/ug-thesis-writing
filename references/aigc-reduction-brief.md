# AIGC 降重快速参考

## 一、核心原则

**三条铁律**:
1. 绝不用 AI 全文重写（重写反而增加 AI 痕迹）
2. 修改率必须超过 40%
3. 确定性替换，非 LLM 重写

**为什么不能用 AI 重写**:
- AI 模型总是选择最高概率的 token，导致困惑度(perplexity)更低
- 低困惑度 + 均匀句式 = 检测器高置信度标记
- 实测：用 Claude 全文重写后，PaperPure 检测率从 35% 升至 100%

## 二、操作顺序

```
扫描定位 → 词汇替换(10-15%) → 句式重构(15-20%) → 段落调整(10-15%) → 验证修改率
```

## 三、词汇替换表（高频词）

### 连接词替换

| 原词 | 替换方案（选一） |

| 其次 | 第二步 / 随后 / 删除 |
| 最后 | 第三步 / 最终 / 删除 |

## 四、句式重构技巧

### 1. 语序重组
- 原句: "本研究采用随机森林模型对分类结果进行预测"
- 修改: "分类预测任务中，本研究选用了随机森林模型"

### 2. 长句拆分
- 原句: "通过对多个样本多年的时序监测数据进行分析，发现目标变量受多个特征因素共同影响"
- 修改: "数据覆盖多个样本，时间跨度为多年，尺度为月。分析结果显示，目标变量受多类因素共同驱动"

### 3. 被动变主动
- 原句: "数据被划分为训练集和测试集"
- 修改: "本研究将数据划分为训练集和测试集"

### 4. 主语替换
- 原句: "结果表明该模型表现最优"
- 修改: "该模型在各项指标上均优于其他对比方法"

### 5. 并列结构打散
- 原句: "该方法具有精度高、速度快、稳定性好的优点"
- 修改: "精度方面，该方法的R²达到0.92；运行效率上，单次预测耗时仅0.3秒；多次重复实验的标准差为0.02"

## 五、段落级调整

### 打破段落对称性
- 相邻段落长度要有差异（不要都是4-5句）
- 不同段落使用不同的开头方式

### 注入实验细节
- 补充参数设置的具体数值
- 添加运行环境信息
- 描述数据预处理的具体步骤

### 注入作者判断
- "笔者认为..."
- "基于实际运行经验..."
- "从工程应用角度看..."

## 六、检测平台特点

| 平台 | 特点 | 建议 |
|------|------|------|
| PaperPure | 极敏感，全AI重写=100% | 必须用确定性替换 |
| 知网3.0 | 98.6%准确率 | 关注模板句式 |
| 维普 | 近期检测更严格 | 提前检测 |
| PaperYY | 可能故意抬高AI率 | 仅作参考 |
| 腾讯朱雀 | 免费 | 适合快速验证 |
